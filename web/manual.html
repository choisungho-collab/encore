메뉴얼

켜두면 알아서 녹화하고 분석하는 브루드워 전적 아카이브.

어떻게 작동하나요

프로그램을 실행해 두면 백그라운드에서 스타크래프트를 감지합니다. 게임을 시작하면 실제 화면을 HD로 녹화하고, 한 판이 끝나면(리플레이 자동저장 감지) 녹화를 멈추고 리플레이를 분석해 갤러리에 한 장의 카드로 올립니다.

  • 영상 = 실제 게임 화면(리마스터 HD). OBS 같은 별도 프로그램 필요 없음.
  • 분석 = 빌드오더 · 유닛 구성 · APM/EAPM · 타운홀 타이밍 · 하이라이트.
  • 리플레이(.rep) = 원본도 같이 보관돼서 다운로드 가능.
실행하면 뜨는 상태 창

콘솔(검은 창) 대신 다크 테마 상태 창이 떠요. 지금 ● 녹화 중인지 대기 중인지, 지금까지 모은 경기 수, 사용 중인 인코더(NVENC 등), 실시간 로그가 보이고, 갤러리 열기 · 녹화 폴더 · 종료 버튼이 있습니다. 창을 닫아도 녹화는 계속돼요.

갤러리 · 매치 분석

메인 화면에 경기가 카드로 쌓입니다. 우측 상단에 ● 녹화 중 같은 실시간 상태가 표시돼요. 카드를 누르면 매치 분석 페이지로 들어갑니다.

플레이어 선택

상단 탭에서 선수를 누르면 그 사람의 빌드오더 · 유닛 · APM이 바로 나옵니다. "나" 표시는 본인이에요.

빌드 비교 (전원)

"빌드 비교 (전원)" 버튼을 누르면 같은 게임 모든 선수의 빌드오더를 한 화면에 나란히 놓고 비교할 수 있어요(가로 스크롤).

하이라이트

영상 아래 하이라이트 칩을 누르면 영상의 해당 지점으로 바로 이동합니다. 기준은:

  • 교전 추정 — 전원의 분당 활동량(APM)이 치솟은 구간 (실제 전투와 다를 수 있는 추정값)
  • 테크 마일스톤 — 러커 · 다크템플러 · 캐리어 같은 핵심 유닛의 첫 등장
  • 첫 확장 — 가장 빠른 멀티 타이밍

설정 (config.json)

폴더 안 config.json 을 메모장으로 열어 바꾼 뒤 프로그램을 다시 켜면 적용됩니다.

항목값설명
encoderauto / nvenc / x264auto: NVENC 되는지 테스트 후 자동 선택. 강제하려면 nvenc(하드웨어)·x264(소프트웨어).
scaleauto / source / 1080 / 720 / 480소프트웨어 인코딩일 때 게임 끊김 방지로 auto가 720p로 낮춤. 화질↑은 1080/source, 더 가볍게는 480.
fps30녹화 프레임. 게임이 버벅이면 20이나 15로.
captureauto / wgc / ddagrab / gdigrabauto는 전체화면도 잡히는 WGC를 우선 사용. 보통 건드릴 필요 없음.
username내 스타 아이디업로더 표시용.

자주 묻는 것

게임이 끊기거나 버벅여요.

소프트웨어 인코딩 중일 가능성이 커요. NVIDIA 드라이버를 최신으로 올리면 NVENC(하드웨어)로 잡혀서 부담이 줄어요. 그래도 무거우면 config.json 에서 "scale":"480", "fps":20 으로 낮추세요.

검은 화면으로 녹화돼요.

전체화면(독점) 모드일 때 옛 방식에서 생기는 문제예요. 기본 WGC 캡처는 전체화면도 잡지만, 안 되면 config.json 의 "capture" 를 "wgc" 로 두거나 스타를 '창(전체 화면)' 모드로 해보세요.

녹화가 중간에 끊겼다가 다시 시작돼요.

캡처가 끊기면 자동으로 복구합니다. 자주 발생하면 캡처 방식을 "wgc" 로 고정해 보세요.

콘솔(검은 창) 없이 쓰고 싶어요.

START_no_console.vbs 로 실행하면 창 없이 조용히 켜지고, 상태는 갤러리 상단에서 확인할 수 있어요.
